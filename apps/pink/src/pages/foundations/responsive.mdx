## Breakpoints

In the Appwrite console we use four breakpoints:

| Scss Variable | Screen Size    | Range             |
| ------------- | -------------- | ----------------- |
| `$break1`     | Small          | 0px - 767px       |
| `$break2`     | Medium         | 768px - 1198px    |
| `$break2open` | Medium (open)  | 768px - Infinite  |
| `$break3open` | Large (open)   | 1199px - infinite |

## Special Use Cases

In some use cases we might want to affect only a part of screen sizes and not all of them. In Pink Desing responsive library, there is no overlap between the different resolusions. As shown in the example below, the style of the small and large screens won't affect the medium screen, and vice versa.

<br />

<div class="box u-stretch" style="padding: 1rem; padding-right: 2.5rem;">
    <CodePreview
        client:load
        code={`
    <style>
        @media #{$break1},  #{$break3open} { } /* small & large screens */
        @media #{$break2} { } /* only medium screens */
    </style>
`}
        language="html"
        lineNumbers
    />
</div>

<br />

       padding: var(--p-partial-padding);
