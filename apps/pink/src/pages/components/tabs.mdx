# Tabs

`tabs`

### Tab navigation allows users to switch between multiple sections of content within a single screen.

## Components

Tabs consist of four components:

| Class                | Type          |             |
| -------------------- | ------------- | ----------- |
| `tabs-list`          | List          | -           |
| `tabs-item`          | Item          | -           |
| `tabs-button`        | Button        | -           |
| `tabs-button-scroll` | Scroll        | Used when there is not enough space to display all tabs, mostly on mobile devices. The `tabs-button-scroll` element can be hidden by adding `u-hide` class to it.             |

## Usage

<br />

In most scenarios, you should use no more than six tabs. This maintains an uncluttered UI and reduces cognitive load for users.

<br />

![3 tabs](/images/tabs-1.svg)

<br />
<br />

If more than six tabs are needed, and you canâ€™t use another patterns in order to keep consistency, we recommend that you add a **horizontal scrolling element** to the tab navigation.

<br />

The right arrow should always be visible. If the user can scroll right, the arrow is active, otherwise the arrow should be disabled. The left arrow should only visible if the user can scroll left.
